## Overview

Create a new cricket match by specifying the competing teams, match format, and configuration details. This endpoint initializes a match in `pending` status, ready for team registration.

## Authentication

This endpoint requires authentication. Include a Bearer token in the Authorization header.

```bash
Authorization: Bearer YOUR_ACCESS_TOKEN
```

## Request Body

### Required Parameters

<ParamField path="teamA" type="string" required>
  Team A ID (MongoDB ObjectId). The first team in the match.
</ParamField>

<ParamField path="teamB" type="string" required>
  Team B ID (MongoDB ObjectId). The second team in the match. Must be different from Team A.
</ParamField>

<ParamField path="title" type="string" required>
  Match title. Example: "Codefied vs Techvaders" or "Team A vs Team B - Quarter Final"
</ParamField>

<ParamField path="startdatetime" type="string" required>
  Match start date and time in ISO 8601 format. If set to a future date, the match will be marked as scheduled.
  
  Example: `"2025-08-01T18:00:00.000Z"`
</ParamField>

<ParamField path="overs" type="integer" required>
  Number of overs per innings. Must be at least 1.
  
  - Test: Set to high value like 450 (unlimited)
  - ODI: Must be 50
  - T20: Must be 20
  - Limited Overs: Any value (e.g., 5, 10, 15)
</ParamField>

<ParamField path="matchType" type="string" required>
  Type of cricket match format.
  
  **Options:**
  - `test` - Test match (2-5 days)
  - `odi` - One Day International (50 overs)
  - `t20` - Twenty20 (20 overs)
  - `limitedOvers` - Custom limited overs format
</ParamField>

<ParamField path="noOfDays" type="integer" required>
  Number of days for the match.
  
  - Test: 2-5 days
  - ODI/T20/Limited Overs: Must be 1
</ParamField>

<ParamField path="oversPerBowler" type="integer" required>
  Maximum overs a single bowler can bowl. Must be 0 or greater.
  
  - Test: 0 (unlimited)
  - ODI: 10
  - T20: 4
  - Limited Overs: Cannot exceed total overs
</ParamField>

### Optional Parameters

<ParamField path="location" type="string">
  Match venue/location. Example: "Karachi", "Lord's Cricket Ground"
</ParamField>

<ParamField path="matchCategory" type="string">
  Match category or type. Example: "friendly", "league", "knockout"
</ParamField>

<ParamField path="ballType" type="string">
  Type of ball used in the match.
  
  **Options:**
  - `leather` - Professional leather ball
  - `tennis` - Tennis ball
  - `other` - Custom ball type
</ParamField>

<ParamField path="pitchType" type="string">
  Type of pitch surface.
  
  **Options:**
  - `rough` - Rough/natural pitch
  - `cement` - Concrete pitch
  - `turf` - Natural grass pitch
  - `astroturf` - Artificial turf
  - `matting` - Matting pitch
  - `other` - Custom pitch type
</ParamField>

<ParamField path="showWagonWheel" type="boolean" default="false">
  Enable wagon wheel visualization for this match. Default is false.
</ParamField>

## Validation Rules

The API enforces ICC-compliant rules:

### Test Match
- `noOfDays` must be between 2 and 5
- `oversPerBowler` must be 0 (unlimited)

### ODI
- `overs` must be exactly 50
- `noOfDays` must be 1
- `oversPerBowler` must be 10

### T20
- `overs` must be exactly 20
- `noOfDays` must be 1
- `oversPerBowler` must be 4

### Limited Overs
- `noOfDays` must be 1
- `oversPerBowler` cannot exceed `overs`

## Error Responses

  ```json
    "code": 400,
    "message": "T20 matches must have 20 overs per innings"
  ```

</ResponseField>

<ResponseField name="401" type="error">

</ResponseField>

<ResponseField name="404" type="error">
  Not Found - One or both teams do not exist

  ```json
  {
    "code": 404,
    "message": "Team A not found"
  }
  ```
</ResponseField>
