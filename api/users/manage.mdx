## Get Users

Retrieve a paginated list of users with optional filtering.

```http
GET /v1/users
```

### Query Parameters

<ParamField query="name" type="string">
  Filter by user name (partial match)
</ParamField>

<ParamField query="role" type="string">
  Filter by role (user, admin)
</ParamField>

<ParamField query="status" type="string">
  Filter by account status
</ParamField>

<ParamField query="sortBy" type="string" default="name:asc">
  Sort results in the format `field:direction` (e.g., `createdAt:desc`)
</ParamField>

<ParamField query="limit" type="integer" default={10}>
  Maximum number of results per page (1-100)
</ParamField>

<ParamField query="page" type="integer" default={1}>
  Page number to retrieve
</ParamField>

<ParamField query="statusFilter" type="string" default="not_deleted">
  Status filter: `all`, `not_deleted`, `active`
</ParamField>

### Response

<ResponseField name="results" type="array">
  Array of user objects
</ResponseField>

<ResponseField name="page" type="integer">
  Current page number
</ResponseField>

<ResponseField name="limit" type="integer">
  Results per page
</ResponseField>

<ResponseField name="totalPages" type="integer">
  Total number of pages
</ResponseField>

<ResponseField name="totalResults" type="integer">
  Total number of matching users
</ResponseField>

<CodeGroup>
```bash cURL
curl -X GET 'https://api.cricpr.com/v1/users?page=1&limit=10&sortBy=createdAt:desc' \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.cricpr.com/v1/users?page=1&limit=10', {
  headers: {
    'Authorization': `Bearer ${accessToken}`
  }
});
const data = await response.json();
```
</CodeGroup>

## Get User

Retrieve a specific user by ID. Users can fetch their own information, admins can fetch any user.

```http
GET /v1/users/:userId
```

### Path Parameters

<ParamField path="userId" type="string" required>
  User ID
</ParamField>

### Query Parameters

<ParamField query="statusFilter" type="string" default="not_deleted">
  Status filter: `all`, `not_deleted`, `active`
</ParamField>

### Response

Returns a complete user object including roles and permissions.

<CodeGroup>
```bash cURL
curl -X GET https://api.cricpr.com/v1/users/507f1f77bcf86cd799439011 \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const response = await fetch(`https://api.cricpr.com/v1/users/${userId}`, {
  headers: {
    'Authorization': `Bearer ${accessToken}`
  }
});
const user = await response.json();
```
</CodeGroup>

## Update User

Update user profile information. Users can update their own profile, admins can update any user.

```http
PATCH /v1/users/:userId
```

### Path Parameters

<ParamField path="userId" type="string" required>
  User ID to update
</ParamField>

### Request Body

<ParamField body="name" type="string">
  User's full name
</ParamField>

<ParamField body="phone" type="string">
  Phone number with country code (e.g., "+1234567890")
</ParamField>

<ParamField body="city" type="string">
  City of residence
</ParamField>

<ParamField body="dob" type="string">
  Date of birth (YYYY-MM-DD format)
</ParamField>

<ParamField body="battingStyle" type="string">
  Batting style: `Left-hand bat`, `Right-hand bat`, or `none`
</ParamField>

<ParamField body="bowlingStyle" type="string">
  Bowling style options:
  - `Right-arm fast`
  - `Right-arm medium`
  - `Left-arm fast`
  - `Left-arm medium`
  - `Slow left-arm orthodox`
  - `Slow left-arm chinaman`
  - `Right-arm Off Break`
  - `Right-arm Leg Break`
  - `none`
</ParamField>

<ParamField body="playingRole" type="string">
  Playing role options:
  - `Top-order batter`
  - `Middle-order batter`
  - `Opening batter`
  - `Lower-order batter`
  - `Wicket-keeper batter`
  - `Wicket-keeper`
  - `Bowler`
  - `All-rounder`
  - `none`
</ParamField>

### Response

<ResponseField name="user" type="object">
  Updated user object with all profile fields
</ResponseField>

<ResponseField name="playerProfile" type="object">
  Updated player profile (if cricket-related fields were changed)
</ResponseField>

<CodeGroup>
```bash cURL
curl -X PATCH https://api.cricpr.com/v1/users/507f1f77bcf86cd799439011 \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Doe",
    "city": "Karachi",
    "dob": "1990-01-01",
    "battingStyle": "Right-hand bat",
    "bowlingStyle": "Right-arm medium",
    "playingRole": "All-rounder"
  }'
```

```javascript JavaScript
const response = await fetch(`https://api.cricpr.com/v1/users/${userId}`, {
  method: 'PATCH',
  headers: {
    'Authorization': `Bearer ${accessToken}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'John Doe',
    city: 'Karachi',
    battingStyle: 'Right-hand bat',
    bowlingStyle: 'Right-arm medium',
    playingRole: 'All-rounder'
  })
});
const { user, playerProfile } = await response.json();
```
</CodeGroup>

## Create User

Create a new user account. Admin-only endpoint.

```http
POST /v1/users
```

### Request Body

<ParamField body="name" type="string" required>
  User's full name
</ParamField>

<ParamField body="email" type="string" required>
  Email address (must be unique)
</ParamField>

<ParamField body="password" type="string" required>
  Password (minimum 8 characters, at least one letter and one number)
</ParamField>

<ParamField body="role" type="string" required>
  User role: `user` or `admin`
</ParamField>

### Response

Returns the created user object (password not included).

<CodeGroup>
```bash cURL
curl -X POST https://api.cricpr.com/v1/users \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Jane Smith",
    "email": "jane@example.com",
    "password": "password123",
    "role": "user"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.cricpr.com/v1/users', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${adminToken}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Jane Smith',
    email: 'jane@example.com',
    password: 'password123',
    role: 'user'
  })
});
const user = await response.json();
```
</CodeGroup>

## Delete User

Permanently delete a user account. Users can delete their own account, admins can delete any user.

```http
DELETE /v1/users/:userId
```

### Path Parameters

<ParamField path="userId" type="string" required>
  User ID to delete
</ParamField>

### Response

Returns `204 No Content` on successful deletion.

<CodeGroup>
```bash cURL
curl -X DELETE https://api.cricpr.com/v1/users/507f1f77bcf86cd799439011 \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const response = await fetch(`https://api.cricpr.com/v1/users/${userId}`, {
  method: 'DELETE',
  headers: {
    'Authorization': `Bearer ${accessToken}`
  }
});
// 204 No Content
```
</CodeGroup>

## Anonymize Account

Anonymize your own user account by removing personally identifiable information. This is a self-service endpoint.

```http
POST /v1/users/anonymize
```

### Response

Returns the anonymized user object.

<CodeGroup>
```bash cURL
curl -X POST https://api.cricpr.com/v1/users/anonymize \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.cricpr.com/v1/users/anonymize', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${accessToken}`
  }
});
const user = await response.json();
```
</CodeGroup>

<Warning>
  Anonymization is irreversible. This action removes your name, email, phone, and other personal details while preserving cricket statistics and match history.
</Warning>

## Error Responses

<Accordion title="400 Bad Request">
  **Duplicate Email**: Email address already exists
  
  ```json
  {
    "code": 400,
    "message": "Email already taken"
  }
  ```
</Accordion>

<Accordion title="401 Unauthorized">
  **Missing or Invalid Token**: Authentication required
  
  ```json
  {
    "code": 401,
    "message": "Please authenticate"
  }
  ```
</Accordion>

<Accordion title="403 Forbidden">
  **Insufficient Permissions**: User lacks required permissions
  
  ```json
  {
    "code": 403,
    "message": "Forbidden"
  }
  ```
</Accordion>

<Accordion title="404 Not Found">
  **User Not Found**: User ID does not exist
  
  ```json
  {
    "code": 404,
    "message": "User not found"
  }
  ```
</Accordion>
