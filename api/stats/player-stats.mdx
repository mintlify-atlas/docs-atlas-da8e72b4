## Update All Match Stats

Update statistics for all active matches (in progress or completed).

<CodeGroup>
```bash cURL
curl -X POST \
  'https://api.cricpr.com/v1/stats/update-all' \
  -H 'Authorization: Bearer YOUR_TOKEN'
```

```javascript JavaScript
const response = await fetch('https://api.cricpr.com/v1/stats/update-all', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});
```
</CodeGroup>

### Response

<ResponseField name="totalMatches" type="number">
  Total number of matches processed
</ResponseField>

<ResponseField name="results" type="array">
  Array of match update results
  <Expandable title="Result Object">
    <ResponseField name="matchId" type="string">
      Match ID
    </ResponseField>
    <ResponseField name="title" type="string">
      Match title
    </ResponseField>
    <ResponseField name="status" type="string">
      Update status: "success" or "error"
    </ResponseField>
    <ResponseField name="error" type="string" optional>
      Error message if status is "error"
    </ResponseField>
  </Expandable>
</ResponseField>

## Update Match Stats

Update statistics for a specific match and all its innings.

<CodeGroup>
```bash cURL
curl -X POST \
  'https://api.cricpr.com/v1/stats/match/60f7b3b3b3f3b3b3b3f3b3b3/update' \
  -H 'Authorization: Bearer YOUR_TOKEN'
```

```javascript JavaScript
const matchId = '60f7b3b3b3f3b3b3b3f3b3b3';
const response = await fetch(`https://api.cricpr.com/v1/stats/match/${matchId}/update`, {
  method: 'POST',

    'Authorization': 'Bearer YOUR_TOKEN'

});
```
</CodeGroup>

## Get Player Match Stats

Retrieve statistics for a specific player in a match.

<CodeGroup>

```bash cURL
  'https://api.cricpr.com/v1/stats/match/60f7b3b3b3f3b3b3b3f3b3b3/player/60f7b3b3b3f3b3b3b3f3b3b4' \
  -H 'Authorization: Bearer YOUR_TOKEN'
```

```javascript JavaScript
const matchId = '60f7b3b3b3f3b3b3b3f3b3b3';
const playerId = '60f7b3b3b3f3b3b3b3f3b3b4';
const response = await fetch(`https://api.cricpr.com/v1/stats/match/${matchId}/player/${playerId}`, {
  headers: {
```

## Get Match Player Stats

Retrieve statistics for all players in a match.

<CodeGroup>
```bash cURL
curl -X GET \
  'https://api.cricpr.com/v1/stats/match/60f7b3b3b3f3b3b3b3f3b3b3/players' \
  -H 'Authorization: Bearer YOUR_TOKEN'
```

```javascript JavaScript
const matchId = '60f7b3b3b3f3b3b3b3f3b3b3';
const response = await fetch(`https://api.cricpr.com/v1/stats/match/${matchId}/players`, {
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});
```
</CodeGroup>

### Path Parameters

<ParamField path="matchId" type="string" required>
  The ID of the match
</ParamField>

### Response

Returns an array of player statistics objects. Each object has the same structure as the "Get Player Match Stats" endpoint response.

## Statistics Calculation Details

### Batting Statistics Calculation

- **Balls Faced**: Legal deliveries + no-balls (wides don't count)
- **Strike Rate**: (runs scored / balls faced) × 100
- **Average**: Runs scored ÷ times dismissed (null if not out)
- **Milestones**: 
  - Hundreds: >= 100 runs
  - Fifties: 50-99 runs
  - Thirties: 30-49 runs
  - Ducks: 0 runs when dismissed

### Bowling Statistics Calculation

- **Overs Bowled**: (balls bowled ÷ 6) + (balls % 6 ÷ 10)
- **Economy Rate**: Runs conceded ÷ overs bowled
- **Runs Conceded**: Includes wides and no-balls (not byes/leg-byes)
- **Maiden Over**: 6 legal deliveries with 0 runs conceded
- **Dot Balls**: Legal deliveries with 0 runs scored

### Fielding Statistics Calculation

- **Catches**: Direct catches by fielder
- **Stumpings**: Credited to wicket-keeper for stumped dismissals
- **Run Outs**: Fielder involved in run-out dismissal

## Error Handling

<ResponseField name="statusCode" type="number">
  HTTP status code (404, 500, etc.)
</ResponseField>

<ResponseField name="message" type="string">
  Error message describing what went wrong
</ResponseField>

Common errors:
- **404**: Match, innings, or player not found
- **401**: Authentication required
- **403**: Insufficient permissions
- **500**: Internal server error during calculation

See [Error Handling](/api/overview#error-handling) for more details.
