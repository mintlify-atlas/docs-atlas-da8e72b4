## Get Player Stats

<api-endpoint method="GET" path="/v1/players/{playerId}/stats" />

Get comprehensive player statistics including batting, bowling, and fielding performance.

### Path Parameters

<ParamField path="playerId" type="string" required>
  Player ID
</ParamField>

### Response

<ResponseField name="player" type="object">
  Player object with basic information
</ResponseField>

<ResponseField name="totalMatches" type="integer">
  Total matches played
</ResponseField>

<ResponseField name="totalRuns" type="integer">
  Total runs scored across all matches
</ResponseField>

<ResponseField name="totalWickets" type="integer">
  Total wickets taken across all matches
</ResponseField>

<ResponseField name="totalCatches" type="integer">
  Total catches taken
</ResponseField>

<ResponseField name="battingStats" type="array">
  Array of batting statistics objects
  
  <Expandable title="Batting stats object">
    <ResponseField name="matchId" type="string">
      Match identifier
    </ResponseField>
    <ResponseField name="runs" type="integer">
      Runs scored in the match
    </ResponseField>
    <ResponseField name="ballsFaced" type="integer">
      Balls faced
    </ResponseField>
    <ResponseField name="fours" type="integer">
      Number of fours hit
    </ResponseField>
    <ResponseField name="sixes" type="integer">
      Number of sixes hit
    </ResponseField>
    <ResponseField name="strikeRate" type="number">
      Strike rate (runs per 100 balls)
    </ResponseField>
    <ResponseField name="isOut" type="boolean">
      Whether the player got out
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="bowlingStats" type="array">
  Array of bowling statistics objects
  
  <Expandable title="Bowling stats object">
    <ResponseField name="matchId" type="string">
      Match identifier
    </ResponseField>
    <ResponseField name="overs" type="number">
      Overs bowled
    </ResponseField>
    <ResponseField name="maidens" type="integer">
      Maiden overs bowled
    </ResponseField>
    <ResponseField name="runs" type="integer">
      Runs conceded
    </ResponseField>
    <ResponseField name="wickets" type="integer">
      Wickets taken
    </ResponseField>
    <ResponseField name="economy" type="number">
      Economy rate (runs per over)
    </ResponseField>
    <ResponseField name="wides" type="integer">
      Wide balls bowled
    </ResponseField>
    <ResponseField name="noBalls" type="integer">
      No balls bowled
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="fieldingStats" type="array">
  Array of fielding statistics objects
  
  <Expandable title="Fielding stats object">
    <ResponseField name="matchId" type="string">
      Match identifier
    </ResponseField>
    <ResponseField name="catches" type="integer">
      Catches taken
    </ResponseField>
    <ResponseField name="runOuts" type="integer">
      Run-outs executed
    </ResponseField>
    <ResponseField name="stumpings" type="integer">
      Stumpings (for wicket-keepers)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="playerMatchStats" type="array">
  Array of per-match statistics objects combining all stats
</ResponseField>

## Get Players by Role

<api-endpoint method="GET" path="/v1/players/role/{role}" />

Get all players filtered by their primary playing role.

### Path Parameters

<ParamField path="role" type="string" required>
  Playing role (e.g., "Batsman", "Bowler", "All-rounder", "Wicket-keeper")
</ParamField>

### Query Parameters

<ParamField query="statusFilter" type="string" default="not_deleted">
  Status filter: `active`, `not_deleted`, or `all`
</ParamField>

### Response

Returns array of player objects matching the role.

## Cricket Statistics Explained

### Batting Metrics

**Runs** - Total runs scored by the batsman

**Balls Faced** - Number of deliveries faced

**Strike Rate** - Runs scored per 100 balls: `(runs / balls_faced) × 100`

**Fours** - Boundaries hit along the ground (4 runs each)

**Sixes** - Boundaries hit over the boundary (6 runs each)

### Bowling Metrics

**Overs** - Number of overs bowled (1 over = 6 legal deliveries)

**Maidens** - Overs with no runs conceded

**Wickets** - Number of batsmen dismissed

**Economy Rate** - Runs conceded per over: `runs / overs`

**Wides** - Deliveries bowled outside the batsman's reach

**No Balls** - Illegal deliveries (front-foot, height, etc.)

### Fielding Metrics

**Catches** - Number of catches taken

**Run Outs** - Direct hits or assisted run-outs

**Stumpings** - Dismissals by wicket-keeper when batsman is out of crease

## Common Use Cases

### Find All Fast Bowlers

```bash
# Right-arm fast bowlers
curl "https://api.cricpr.com/v1/players/bowling-style/Right-arm%20fast"

# Left-arm fast bowlers
curl "https://api.cricpr.com/v1/players/bowling-style/Left-arm%20fast"
```

### Find All-Rounders

```bash
curl https://api.cricpr.com/v1/players/role/All-rounder
```

### Get Player Career Statistics

```bash
curl https://api.cricpr.com/v1/players/PLAYER_ID/stats
```

### Filter Active Left-Handed Batsmen

```bash
curl "https://api.cricpr.com/v1/players/batting-style/Left-hand%20bat?statusFilter=active"
```

## Performance Analysis

Use the statistics endpoint to analyze:

- **Best batsmen** - Sort by totalRuns or batting average
- **Top bowlers** - Sort by totalWickets or economy rate  
- **Best fielders** - Sort by totalCatches
- **Match winners** - Identify all-rounders with high runs and wickets
- **Form analysis** - Recent match statistics trends

### Example: Finding Top Performers

```bash
# Get player stats for team analysis
curl https://api.cricpr.com/v1/players/PLAYER_1/stats
curl https://api.cricpr.com/v1/players/PLAYER_2/stats
curl https://api.cricpr.com/v1/players/PLAYER_3/stats

# Compare totalRuns, totalWickets, and match performance
```
